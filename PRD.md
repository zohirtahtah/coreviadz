# وثيقة متطلبات المنتج (PRD) - Corevia DZ

## 1. ملخص تنفيذي
**Corevia DZ** هو نظام ERP متكامل ومتعدد المستأجرين (SaaS) لإدارة شركات الخياطة والملابس في الجزائر. يوفر النظام إدارة شاملة للطلبيات، المنتجات، المخزون، الموردين، العمال، المصروفات، والأرباح، مع دعم كامل للغة العربية وخرائط الولايات الجزائرية.

---

## 2. نظرة عامة على المنتج

### 2.1 الرؤية
توفير منصة ERP بسيطة وقوية ومجانية للشركات الصغيرة والمتوسطة في الجزائر، تعمل دون اتصال بالإنترنت (offline-first) مع مزامنة سحابية اختيارية.

### 2.2 القيمة المقترحة
- **مجاني تماماً** - بدون رسوم اشتراك أو تراخيص
- **يعمل بدون إنترنت** - التخزين المحلي هو المصدر الأساسي للبيانات
- **بالعربية الفصحى** - واجهة كاملة بالعربية مع دعم الفرنسية والإنجليزية
- **سحابي اختياري** - مزامنة مع Supabase عند توفر الإنترنت
- **تثبيت كبرنامج سطح مكتب** - عبر Electron أو PWA

### 2.3 الجمهور المستهدف
| النوع | الوصف |
|---|---|
| المستخدم الأساسي | أصحاب شركات الخياطة والملابس في الجزائر |
| المستخدم الثانوي | الموظفون والعمال في الشركة |
| المستخدم الثالث | مدير النظام (Super Admin) لإدارة المنصة |

---

## 3. الشخصيات المستهدفة (Personas)

### 3.1 صاحب الشركة (Admin)
- **الاحتياجات**: إدارة الطلبيات، المخزون، العمال، الحسابات، الأرباح
- **نقاط الألم**: صعوبة تتبع الطلبيات يدوياً، فقدان البيانات، عدم وجود تقارير واضحة
- **الأهداف**: زيادة الأرباح، تقليل الأخطاء، تحسين إدارة العمال

### 3.2 الموظف (Employee)
- **الاحتياجات**: تقديم تقارير العمل الإضافي والغياب، التواصل مع الإدارة
- **نقاط الألم**: عدم وجود طريقة منظمة لتقديم الطلبات
- **الأهداف**: تسجيل ساعات العمل والمصاريف بسهولة

### 3.3 مدير المنصة (Super Admin)
- **الاحتياجات**: إدارة الشركات المسجلة، مراقبة النشاط
- **الأهداف**: ضمان استقرار المنصة وإدارة الحسابات

## 6. المعمارية التقنية

### 6.1 تكنولوجيا stack
| الطبقة | التقنية |
|---|---|
| الواجهة | React 19 + TypeScript + Vite 6 |
| التصميم | Tailwind CSS v4 |
| الأيقونات | Lucide React |
| الرسوم البيانية | Recharts |
| الخرائط | Leaflet + SVG (Algeria wilayas) |
| الحركة | Motion (Framer Motion) |
| التخزين المحلي | localStorage (عبر storageUtils.ts) |
| السحابة | Supabase (PostgreSQL + Realtime + Auth) |
| سطح المكتب | Electron + electron-builder |
| المزامنة | Google Sheets API (اختياري) |
| i18n | ترجمة يدوية (ملف translations.ts) |

### 6.2 تدفق البيانات
```
[A] إدخال المستخدم ← [React Component]
    ↓
[storageUtils.ts] ← حفظ في localStorage (المصدر الرئيسي)
    ↓
[supabaseSync.ts] ← مزامنة إلى Supabase (في الخلفية)
    ↓
[Supabase Realtime] ← تحديث تلقائي للأجهزة الأخرى
    ↓
[refreshKey] ↑ ← إعادة تحميل المكونات
```

### 6.3 جداول Supabase
1. `corevia_orders` - الطلبيات

6. `corevia_workers` - العمال
7. `corevia_salary_sheets` - كشوف الرواتب
8. `corevia_expenses` / `corevia_ad_expenses` - المصروفات
9. `corevia_companies` - الشركات

## 9. سيناريوهات الاستخدام الرئيسية

### 9.1 تسجيل شركة جديدة
1. المستخدم يفتح التطبيق → صفحة تسجيل الدخول
2. يختار "إنشاء حساب جديد"
3. يدخل البريد الإلكتروني، الاسم، كلمة المرور
4. الحالة تصبح "قيد الانتظار" → ينتظر موافقة المشرف العام
5. بعد الموافقة → يدخل إلى معالج الإعداد (Onboarding)
6. يكمل 4 خطوات → يبدأ باستخدام النظام

### 9.2 دعوة موظف
1. المدخل يذهب إلى Users Permissions
2. يضيف موظف جديد (الاسم، رقم الهاتف، البريد)
3. النظام يولد رابط دعوة (setup_worker=true&id=...&user=...&pass=...)
4. يرسل الرابط للموظف
5. الموظف يفتح الرابط → يسجل دخول تلقائياً ← يرى فقط الصفحات المصرح بها

### 9.3 إضافة طلبية
1. المدخل يختار Orders ← Add Order
2. يدخل بيانات العميل (الاسم، الهاتف، الولاية، البلدية)
3. يضيف منتجات مع الألوان والمقاسات والكميات
4. يحدد سعر التوصيل وشركة التوصيل
5. يحفظ → تظهر في القائمة ويتم خصم الكميات من المخزون

### 9.4 كشف راتب عامل
1. المدخل يختار Workers ← Salary Sheets
2. يختار الشهر والسنة
3. النظام يحسب تلقائياً: الراتب الأساسي - الغياب + السلف + المكافآت - الخصومات
4. يمكن طباعة أو حفظ الكشف

### 9.5 مزامنة بين الأجهزة
1. المستخدم يضيف طلبية على الجهاز A (تحفظ في localStorage + Supabase)
2. Supabase Realtime يرسل تحديث
3. الجهاز B يتلقى التحديث عبر subscription
4. refreshKey يزيد → المكونات تعيد تحميل البيانات

| المقياس | الهدف | طريقة القياس |
